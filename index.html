WELCOME TO WORLD OF CODE-MIXING

Aim

Code-mixing, the alternation of two or more languages within a single utterance or sentence, presents unique challenges and opportunities for linguistic analysis and natural language processing. Our project proposal aims to rectify the aforementioned constraints by the curation and annotation of extensive Hindi-English material that is blended with Indic codes. Additionally, for the benefit of the NLP community interested in code-mixing research, we hope to provide a number of top-notch low-level NLP tagging tools that do named entity recognition, token-level language identification, and POS tagging. In order to successfully train the recent DL architectures with the increasing popularity of DL-based machine translation (MT) systems that generate high-quality translations in many European, English, and Mandarin languages, we aim to create new datasets that are at least 100 times larger than the previous datasets, if any existed at all.We also see a comparable focus on code-mixed translation. Nevertheless, we are not aware of any study that curates large-scale parallel triplets of code-mixed sentences and the accompanying monolingual translations, with the exception of a few recent efforts. We intend to manually translate each code-mixed text to its equivalent English and Hindi language in addition to the token-level language tags. Finally, we seek to create topical and matrix language predictors, since our proposed dataset also includes topic and matrix language (sentence follows the grammatical structure of the matrix language) for each instance.

TASK

How work done in different tasks


The field of Natural Language Processing, a subfield of Artificial Intelligence, is concerned with providing computers with the ability to comprehend, decipher and create Human Language. Its applications span a wide range of domains, from language translation and sentiment analysis to virtual assistants and chatbots.

NLP tasks such as Language Identification (LID), Named Entity Recognition (NER), and Part-of-Speech (POS) Tagging are essential for comprehending and analyzing text data in various contexts:

Language Identification (LID)

Language Identification is the process of determining the language of a given piece of text. This is essential for a variety of applications such as Multilingual Content Processing, Language Specific Analysis and Internationalization of Software. With an accurate LID, systems can automatically adapt to the input language of the user, enabling a smoother user experience and more effective communication across diverse linguistic contexts.
We can solve the sentence-level Language Identification by identifying the matrix language of the code-mixed text.

Matrix Language Identification Example:

In our approach to Matrix language identification, we annotate each sentence with the language tag that corresponds to its grammatical structure. For instance:
Hindi Sentence: "Are cricket se sanyas le liya kya viru aur social service suru"
Tagged Matrix Language: Hindi
English Sentence: "I love playing lukka chuppi with my friends"
Tagged Matrix Language: English

Token-level language annotation:

In this annotation scheme, we annotate each token of the sentences to its corresponding language tag. The following example shows a token-level language annotation:

Sentence: Are  cricket  se  sanyas  le  liya  kya  viru  aur  social  service  suru
Lang tag:  Hin   Eng    Hin   Hin   Hin  Hin  Hin  IN   Hin    Eng     Eng      Hin

Here, Hin, Eng and IN represent Hindi, English and Language independent tokens, respectively.

Named Entity Recognition (NER):

Named Entity Recognition is a natural language processing task that identifies and categorizes named entities within text into predefined categories such as names of persons, organizations, locations, dates, and more. NER is essential to the extraction of information, search engines, question answering systems, social media monitoring and other applications where identifying specific entities is a prerequisite for understanding and analyzing.

In Named Entity Recognition we perform the token level entity by annotating named entities in the given sentence by using standard English named entities:
i. Person: Modi, Rahul
ii. Location: Himalaya, Ganga
iii. Organization: Infosys, IIT
iv. Date/Time: Today, tomorrow, kal, aaj
v. Geopolitical Entities: India, China, US, Bharat

In addition to the above, we aim to also tag social-media-based specific entities like:
I. Hashtags
II. Mentions
III. Emoji
The named entities are language-independent and require special care during the post-processing stages of any code-mixed task.

Part-of-Speech (POS) Tagging:

Part-of-Speech Tagging is the process of assigning grammatical categories (such as noun, verb, adjective, etc.) to words in a text based on their context and definition. POS tagging is a fundamental task in natural language processing, serving as the foundation for many higher-level language understanding tasks. It facilitates tasks such as text analysis, sentiment analysis, and machine translation by allowing machines to understand the syntactic structure of sentences.

In POS tagging we annotate the given tags. Overall, the plan is to include language-specific POS tags. For example, in a Hinglish sentence, if the token belongs to Hindi, the corresponding Hindi POS tag will be assigned, and if the token belongs to English, the corresponding English POS tag will be assigned.

Annotation Framework

How work done in Annotation


We intend to create a web-based annotation framework in place of the unconventional spreadsheet-based annotation method. By providing helpful options in the dropdown menu or by presenting the relevant fields based on the previous selection, it will speed up the annotation process and make sentences easier to annotate overall.

Commentator is the name of the code-mixed multilingual text annotation framework we present, specifically built for code-mixed text. In the annotation framework of the proposed system there are primarily two users: i) the annotators and (ii) the admins. The annotators perform the annotation task. Whereas the admins design the annotation task, employ annotators, administer the annotation task, and process the annotations. The admins and the annotators need to signup and login to access the various functionalities of the tool.Given these roles, we describe COMMENTATOR functionalities by introducing the two user panels:

The Annotator Panel: The annotator panel contains two pages:

1. Annotation page: The below figure illustrates the annotation page. Given a candidate sentence, it allows two levels of annotations. At the first level, the annotator can choose the matrix language of the sentence (see point 1). Out of two supported languages, one can be selected. At the second level, the annotator chooses a language tag for each token of the sentence (see point 2). To make the annotation process faster, we pre-assign each token with a language tag using a state-of-the-art language identification tool. The annotator can update a pre-assigned tag by clicking on the respective tag button. The major annotation instructions are on the left side of the annotator page (see point 3). In below figure, we present a candidate Hinglish sentence to an annotator with username user1. The annotator can select one out of the three possible language tags, Hindi (shown in Yellow color), English (shown in Green color), or language indepen- dent (shown in Blue color). The selection is made by clicking the same tag button sequentially until the desired tag appears. If the annotator incorrectly annotates a sentence, they can update their annotations by clicking on the “Edit Annotations” button (see point 4), which redirects them to the history and edit page.

2. History and edit page: The below figure showcases history and edit page. It contains a list of sentences annotated in the past along with the timestamp. The annotator can click on any sentence to edit the annotations. A click on a sentence opens the annotation page with previously chosen annotation tags.

The Admin Panel:The below figure shows the admin panel. The admin panel performs three major tasks:

1. Data upload: The administrator can upload the source sentences using a CSV file (see point 1).

2. Annotation analysis: The administrator can: (i) analyze the quality of annotations using Cohen’s Kappa score for inter-annotator agreement (IAA) (see point 3) and (ii) analyze the degree of code-mixing in the annotated text using the code-mixing index (CMI)(see point 2).

3. Data download: The admin can download annotations of single/multiple annotators in a CSV file. The data download functionality also supports the conditional filtering of data based on IAA and CMI.
